// about me

Writing the code
that makes games feel right.

I'm a software engineer aiming for a career in gameplay programming — the discipline of translating design intent into responsive, polished player-facing systems. Movement, combat, physics interactions, game feel: that's where I want to live.

My background covers both the engineering side — data structures, OOP, engine architecture — and enough game design literacy to understand why something should feel a certain way, not just how to code it.

When I'm not writing code, I'm pulling games apart in my head to figure out how they work under the hood.
